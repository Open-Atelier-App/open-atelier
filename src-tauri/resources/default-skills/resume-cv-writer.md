# Resume / CV Writer

When the user asks you to write, update, or tailor a resume or CV, produce something ready to submit — not a generic template with brackets everywhere.

1. Use only what the user actually gave you (past roles, dates, achievements, skills). Never invent employers, titles, dates, or metrics — if a detail is missing, leave an obvious placeholder like `[company name]` rather than guessing.
2. Favor concrete, quantified achievements over vague duty descriptions: "Cut deploy time from 40min to 8min" beats "Responsible for deployments." If the user only gave you a duty, ask them for the outcome/number before inventing one — or leave it as-is rather than fabricating a metric.
3. If the user shares a job posting alongside their background, tailor the summary and bullet order to what that posting emphasizes, without adding skills or experience the user doesn't have.
4. Structure: contact line, brief summary (optional, 1-2 lines), experience in reverse-chronological order with 2-4 bullets per role, education, skills. Keep it to one page's worth of content unless the user's background clearly warrants more.
5. Write the result to a file (CREATE/WRITE) rather than pasting it into chat — this is exactly the kind of substantial, reusable document that belongs in the workspace, not the conversation.
6. If asked for a cover letter instead, match its tone to the role (formal for corporate/legal/finance, warmer for creative/startup roles) and keep it under one page — a short letter that makes a specific case beats a long one that restates the resume.
