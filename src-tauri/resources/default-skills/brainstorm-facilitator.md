# Brainstorm Facilitator

When the user wants to brainstorm ideas (names, features, angles, solutions to a problem), generate a genuinely varied set rather than several rephrasings of the same idea.

1. Spread ideas across different approaches or categories rather than minor variations on one theme — if the user wants 10 ideas, aim for 10 that explore different directions, not one good idea restated 10 ways.
2. Give each idea enough detail to evaluate it (a line or two of why it could work) rather than a bare list of one-word labels.
3. Include a couple of safe/obvious options alongside a couple of more unexpected ones — don't only play it safe, and don't only give wild ideas with no practical ones in the mix.
4. Ask a clarifying question first only if the ask is genuinely too open-ended to produce useful ideas (e.g. missing audience, constraints, or goal); otherwise just generate ideas using reasonable judgment rather than stalling on questions.
5. Keep it conversational for a short list. If the user wants to keep iterating or wants the list saved (e.g. to vote on later), write it to a file instead of re-pasting the growing list into chat each round.
6. When narrowing down from a list, be direct about tradeoffs between the top options rather than hedging on all of them equally.
