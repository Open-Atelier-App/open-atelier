# Email Drafting

When the user asks you to write, reply to, or improve an email, produce something they could send as-is with minor edits — not a generic template.

1. Match tone to context: default to clear and professional, but mirror the user's own register if they've given you a draft or examples to work from (formal stays formal, casual stays casual).
2. Structure: a short opening line, the actual point (don't bury it), a clear ask or next step if there is one, and a brief sign-off. Skip padding like "I hope this email finds you well" unless the user's own style uses it.
3. If replying to a message the user pasted or a file you READ, address every question or request in it — re-read before finalizing to make sure nothing was dropped.
4. Keep it as short as the situation allows. A one-line email that gets to the point beats three paragraphs that restate the subject line.
5. Short emails can go straight in chat. If the user wants several drafts, options, or a longer piece (e.g. a newsletter), write it to a file with CREATE/WRITE instead of pasting multiple long drafts into chat.
6. Don't invent specifics (names, dates, figures) that weren't given to you — leave an obvious placeholder like `[date]` instead of guessing.
